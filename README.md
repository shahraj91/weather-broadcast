# Weather Broadcaster

Daily WhatsApp weather + trivia broadcaster for a global user base.
Sends at 7:30 AM in each user's local timezone via Twilio. Messages are
generated by a local Llama model running through [Ollama](https://ollama.com).
Weather data is free from [Open-Meteo](https://open-meteo.com) — no API key needed.

## What it does

- Looks up each user's local weather at 7:30 AM in their timezone
- Generates a personalised WhatsApp message (weather summary + fun fact) using a local LLM
- Tailors the message based on the user's activity (runner, cyclist, farmer, photographer, parent)
- Sends it via Twilio WhatsApp API
- Evaluates weather against risk thresholds and sends a separate alert if any are triggered
- Responds to inbound WhatsApp messages via a Flask webhook (weather queries, activity updates, unsubscribe)
- Logs every send attempt to SQLite for auditing

Units (°C/°F, km/h/mph) are automatically detected from each user's location.

## Setup

### 1. Prerequisites

- Python 3.12+
- [Ollama](https://ollama.com) installed and running locally
- A Twilio account with a WhatsApp-enabled sender number

### 2. Install dependencies

```bash
python3 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 3. Configure environment

```bash
cp .env.example .env
```

Open `.env` and fill in your credentials (see [Environment Variables](#environment-variables) below).

### 4. Pull a model into Ollama

```bash
ollama pull llama3
```

Or use any model you prefer — set `LLAMA_MODEL` in `.env` accordingly.

### 5. Initialise the database and seed test users

```bash
python3 database/seed.py
```

### 6. Start the scheduler

```bash
python3 main.py
```

The scheduler runs continuously, firing a broadcast job at 7:30 AM for every
IANA timezone that has at least one active user.

## Adding users

Create a CSV file named `users_to_add.csv` in the project root:

```csv
phone,lat,lon,name
+14155552671,37.7749,-122.4194,Alex
+447700900123,51.5074,-0.1278,Charlie
+919876543210,28.6139,77.2090,Priya
```

Then run:

```bash
python3 add_users.py
```

Timezone and unit system are resolved automatically from `lat`/`lon`.

## Environment variables

Copy `.env.example` to `.env` and set the following:

| Variable | Description | Default |
|----------|-------------|---------|
| `TWILIO_ACCOUNT_SID` | Twilio Account SID | *(required)* |
| `TWILIO_AUTH_TOKEN` | Twilio Auth Token | *(required)* |
| `TWILIO_WHATSAPP_FROM` | Twilio WhatsApp sender (e.g. `whatsapp:+14155238886`) | *(required)* |
| `TWILIO_SANDBOX_KEYWORD` | Sandbox join keyword shown in Twilio Console | *(required for sandbox)* |
| `LLAMA_API_URL` | Ollama generate endpoint | `http://localhost:11434/api/generate` |
| `LLAMA_MODEL` | Ollama model name | `llama3` |
| `LLAMA_TIMEOUT` | Ollama request timeout in seconds | `60` |
| `DB_PATH` | SQLite database file path | `./data/weather_broadcast.db` |
| `WEBHOOK_ENABLED` | Start Flask webhook server on launch | `false` |
| `WEBHOOK_PORT` | Port for the Flask webhook server | `5000` |
| `NGROK_ENABLED` | Reminder flag — ngrok must be started manually | `false` |

## Message format

Every outgoing WhatsApp message contains:

1. A personalised greeting + structured weather summary (high/low, condition, wind, humidity)
2. `🌟 Fun Fact:` — one kid-friendly nature or weather fact

Messages are generated by Ollama. If Ollama is unavailable or times out, a static
fallback template is used automatically — no message is ever skipped due to LLM failure.

---

## Risk alerts

After each morning message, the weather is evaluated against six thresholds:

| Condition | Threshold |
|-----------|-----------|
| Extreme heat | Temp high > 35°C / 95°F |
| Dangerous cold | Temp low < −10°C / 14°F |
| Strong winds | Wind > 60 km/h / 37.3 mph |
| Thunderstorm | Condition contains "thunderstorm" |
| Dense fog | Humidity > 90% and foggy |
| Heat index | Temp high > 30°C / 86°F and humidity > 70% |

If any threshold is crossed, a separate `⚠️ Weather Alert` WhatsApp message is sent immediately after the morning broadcast. Logged with `status="risk_alert"`.

## Two-way conversation

With `WEBHOOK_ENABLED=true`, the system responds to inbound WhatsApp messages. Supported intents (auto-classified by Llama):

| Intent | Example message | Action |
|--------|----------------|--------|
| `WEATHER_QUERY` | "Will it rain today?" | Fetches fresh forecast, answers question |
| `ACTIVITY_UPDATE` | "I'm a runner" | Saves activity, tailors future messages |
| `WEATHER_NOW` | "What's the weather now?" | Returns current conditions immediately |
| `UNSUBSCRIBE` | "stop" / "cancel" | Deactivates user |
| `GENERAL` | Anything else | Helpful LLM response |

To expose locally: run `ngrok http 5000` and paste the HTTPS URL + `/webhook` into Twilio Console → Sandbox Settings → "When a message comes in".
