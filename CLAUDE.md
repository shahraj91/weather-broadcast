# Weather Broadcast System

Daily WhatsApp weather + trivia broadcaster for a global user base.
Sends at 7:30 AM in each user's local timezone via Twilio. Messages are
generated by a local Llama model via Ollama. Free weather data from Open-Meteo (no API key).

## Stack
- Python 3.12, SQLite (→ Postgres later), APScheduler, Twilio WhatsApp API
- Local Llama model via Ollama HTTP API for message generation
- Open-Meteo API for weather (free, no key, lat/lon based)
- pytest for all tests


## Key Commands
```bash
# Setup
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env           # then fill in credentials

# Database
python3 database/seed.py       # seed 10 global test users

# Add users — CLI (timezone/units auto-detected from lat/lon)
python3 add_users.py --name "<name>" --phone +<country_code><number> --lat <latitude> --lon <longitude>
python3 add_users.py --phone +<country_code><number> --lat <latitude> --lon <longitude>  # name optional
python3 add_users.py --name "<name>" --phone +<country_code><number> --lat <latitude> --lon <longitude> --opted-in  # already joined sandbox

# Add users — CSV (edit users_to_add.csv first: phone,lat,lon,name)
python3 add_users.py --csv

# List users
python3 list_users.py              # all active users
python3 list_users.py --all        # include inactive
python3 list_users.py <name>       # search by name
python3 list_users.py +<country_code><number> # search by phone

# View send history
python3 list_sends.py              # last 20 sends
python3 list_sends.py --all        # full history
python3 list_sends.py --failed     # only failures
python3 list_sends.py <name>       # sends for a specific user by name
python3 list_sends.py +<country_code><number> # sends for a specific user by phone

# Sandbox opt-in (Twilio sandbox only — required before messages are delivered)
python3 opt_in_user.py --list                   # show opt-in status of all users
python3 opt_in_user.py <name>                   # mark user opted in by name
python3 opt_in_user.py +<country_code><number>  # mark user opted in by phone
python3 migrate_sandbox.py                      # run once on existing DBs to add column

# Manual send (bypasses scheduler — useful for testing)
python3 send_now.py                     # all active timezones
python3 send_now.py <name>              # by user name
python3 send_now.py +<country_code><number>  # by phone number
python3 send_now.py America/Los_Angeles # by timezone

# Run
python3 main.py                # starts blocking scheduler

# Test
pytest tests/ -v               # all unit tests (no network needed)
pytest tests/ -v -m 'not integration'   # skip integration tests
pytest tests/ --cov=. --cov-report=term-missing  # with coverage
```

## Architecture Rules — Do Not Change Without Discussion
- **One APScheduler job per IANA timezone** — never one job per user
- **Llama always runs via local Ollama HTTP API** — do not use llama.cpp subprocess or a remote API
- **Open-Meteo for weather** — free, no key required; do not add a paid API
- **Static fallback in formatter.py** — always present if Ollama fails
- **Unit system is auto-detected from lat/lon** — never ask users to set it
- **WhatsApp phone numbers must be E.164** — enforced in broadcaster.py

## Code Conventions
- All external calls (Twilio, Open-Meteo, Llama, geopy) must be mockable
- Every module has a matching `tests/test_<module>.py`
- Use dataclasses (`User`, `SendLog`) — do not use raw dicts for DB records
- Errors: raise typed exceptions (`WeatherFetchError`, `BroadcasterError`, etc.)
- Logging: use `logging` module, not `print()`
- `.env` values loaded via `python-dotenv` at module top — never hardcode

## Message Format (enforced in formatter.py)
Every outgoing WhatsApp message must contain:
1. Weather summary — 3-4 sentences, 1-2 emojis, mentions high/low/wind
2. `🌟 Fun Fact:` — one kid-friendly nature/weather fact

## Twilio Sandbox Opt-In
The Twilio WhatsApp Sandbox requires every recipient to send a one-time join code
before messages can be delivered. Users who haven't opted in will have their sends
logged as `status="skipped"` (not `failed`) and will not count against retry logic.

Workflow:
1. User sends `join <keyword>` from their WhatsApp to the sandbox number
2. Operator runs `python3 opt_in_user.py <name or phone>` to mark them as opted in
3. Messages will now be delivered at the next scheduled send

`send_now.py` will print a reminder showing unapproved users and the exact message
they need to send whenever a send is triggered and opt-in is missing.

## Environment Variables (see .env.example)
- `TWILIO_ACCOUNT_SID`, `TWILIO_AUTH_TOKEN`, `TWILIO_WHATSAPP_FROM`
- `TWILIO_SANDBOX_KEYWORD` — join keyword shown in Twilio Console sandbox page
- `LLAMA_API_URL` (default: `http://localhost:11434/api/generate`)
- `LLAMA_MODEL` (default: `llama3`)
- `LLAMA_TIMEOUT` (default: `60` seconds)
- `DB_PATH` (default: `./data/weather_broadcast.db`)
